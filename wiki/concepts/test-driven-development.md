# 测试驱动开发（Test-Driven Development, TDD）

## 一句话定义

TDD（测试驱动开发）是一种软件开发方法论，要求先编写测试用例，再编写使其通过的最小实现代码，最后进行重构优化——简称"红-绿-重构"循环。

## 核心循环

```
[红] 写一个会失败的测试 → [绿] 写最少代码使测试通过 → [重构] 优化代码结构 → 重复
```

在 AI 编程语境下，[[OpenSquilla]] 将此循环自动化，扩展为 [[红绿回归测试]]，加入了回归验证和自动修复闭环。
